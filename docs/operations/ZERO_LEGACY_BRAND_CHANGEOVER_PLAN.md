# Zero Legacy Brand Changeover Plan

## Objective

Eliminate every occurrence of the legacy product token from code, paths, configuration, docs, tests, packaging, and CI.

## Non-Negotiable Rules

- No compatibility fallback keys or aliases.
- No dual-prefix environment variable behavior.
- Any legacy token dependency must be logged with exact file/key/path and corrected in the same phase.
- Every phase must reduce both content and path match totals.
- Release gate: zero matches in repository content and repository paths.

## Phased Execution

### Phase 0: Guardrails (in progress)

Scope:

- Remove fallback config binding and dual-prefix runtime behavior.
- Add deterministic audit script and artifact generation.

Exit criteria:

- Runtime rejects legacy configuration namespace.
- Local audit script available and repeatable.

### Phase 1: Configuration and Runtime Surfaces

Scope:

- Eliminate legacy token in env vars, config docs, startup checks, and runtime-facing labels.
- Replace legacy process/service constant names with neutral names.

Exit criteria:

- Zero legacy token references in `.env.example`, startup config code, and runtime config loading.
- Failing startup logs include exact legacy key names for correction.

### Phase 2: API/HTTP Structural Rename

Scope:

- Rename `src/Readarr.Api.V1` and `src/Readarr.Http` directories.
- Rename associated projects, namespaces, imports, and references in solution/build/test assets.

Exit criteria:

- No project, folder, or namespace names include legacy token.
- Build and targeted tests pass with renamed API/HTTP projects.

  - `src/Readarr.Http` -> `src/Bibliophilarr.Http`
- Project files renamed and all solution/project references updated.
- Namespace/import replacement applied for `Readarr.Api.V1` and `Readarr.Http` tokens.

  - path matches: `36`

### Phase 3: Core/Test/Frontend Residuals

Scope:

- Remove residual token usage from core logic, tests, fixtures, localization keys, and frontend assets.
- Rename files and symbols still carrying legacy token.

Exit criteria:

- No content matches in `src/NzbDrone.Core`, `src/NzbDrone.Core.Test`, `src/NzbDrone.Common*`, `frontend/src`.

### Phase 4: Packaging, Distribution, CI, and Tooling

Scope:

- Rename installer/app bundle/file artifacts and pipeline definitions.
- Remove legacy naming from shell scripts, workflows, and packaging metadata.

Exit criteria:

- No path matches in `distribution/`, CI workflows, build scripts, or solution/project files.

### Phase 5: Hard Gate to Zero

Scope:

- Turn on strict enforcement in CI with `--enforce-zero`.
- Block merge on any reintroduction.

Exit criteria:

- `content matches: 0`
- `path matches: 0`
- CI fail-fast on any non-zero reintroduction.

## CI Gate Plan

Current state:

- `rename/*` branches run strict audit mode with `--enforce-zero` in backend CI.
- Non-rename branches run visibility mode and publish audit artifacts.

1. Immediately: run audit in CI and publish artifact (visibility gate).
2. Mid-migration: enforce non-increasing threshold per branch lane.
3. Final: enforce absolute zero with `--enforce-zero`.

## Rollback Policy

- If a phase introduces a runtime break, revert only the phase commit(s).
- Keep audit artifacts from failed attempts for root-cause analysis.
- Do not reintroduce fallback aliases as a rollback shortcut.
